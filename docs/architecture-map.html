React 19 · Vite · Express API · MariaDB

Aktuelle Architektur

React/Vite Frontend
  -> useDb.ts
  -> apiClient.ts
  -> HTTP JSON /api
  -> Express API server/index.js
  -> MariaDB Driver server/db.js
  -> MariaDB tuv_workflow

Frontend

Views, Modale, Tabellen, Charts und Formulare. Keine direkte Datenbankverbindung.

src/views, src/features, src/components

State und HTTP

useDb.ts verwaltet State und optimistische Updates. apiClient.ts kapselt Fetch-Aufrufe.

Express API

server/index.js stellt CRUD-Endpunkte bereit und setzt Workflow-Regeln serverseitig durch.

MariaDB

server/db.js erstellt Tabellen, Constraints und Stammdaten. MariaDB ist die zentrale Persistenz.

Datenfluss

SchrittKomponenteAufgabe
1ViewUser-Aktion ausloesen
2useDbState aktualisieren und API aufrufen
3apiClient.tsHTTP Request an /api senden
4server/index.jsRequest validieren, SQL ausfuehren, JSON mappen
5MariaDBDaten dauerhaft speichern

Wichtige Regeln

  • SQL bleibt im Backend.
  • Frontend nutzt nur /api.
  • MariaDB-Zugangsdaten liegen nur in .env.
  • WF-01 wird in UI und API abgesichert.
  • MariaDB ist die einzige Datenbank dieses Systems.
